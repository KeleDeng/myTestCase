TC-BASE-001 基础配置 新增有效的仓间调拨触发检验配置 1. 进入【基础配置-仓间调拨触发检验配置】页面
2. 新增一条配置：调出仓=“宜宾坯布仓”，调入仓=“盛泽仓”
3. 保存配置		 1. 配置保存成功，提示“保存成功”
2. 配置列表显示新添加的组合		 P0
TC-BASE-002 基础配置 配置不匹配，调拨不触发检验 1. 已配置：A仓→B仓触发检验
2. 创建调拨单：调出仓=A仓，调入仓=C仓
3. 执行调拨入库审核		 1. 审核通过
2. 未生成任何入库检验单
3. 对应入库单的质检状态不为“检验中”		 P1
TC-BASE-003 基础配置 多主体（辰景、新全景、盛景）配置独立生效 1. 在辰景主体下配置：仓1→仓2
2. 在新全景主体下配置：仓3→仓4
3. 切换不同主体账号，分别执行对应配置的调拨入库		 各主体仅对自身配置的仓库组合生效，主体间配置互不影响 P1

二、 调拨入库与检验单生成模块测试

用例ID 模块 测试场景 测试步骤 预期结果 优先级
TC-IN-001 调拨入库审核 审核符合配置的调拨单，自动生成入库检验单 1. 创建一条调拨单，明细包含商品、货主、批号、批次、质量等级等信息
2. 确保调出仓、调入仓组合已在基础配置中
3. 执行调拨入库审核操作		 1. 审核成功
2. 系统自动生成一张或多张入库检验单（验布来源为“调拨入库”）
3. 对应的入库单状态更新为“检验中”		 P0
TC-IN-002 调拨入库审核 按关键维度拆分生成多张检验单 1. 创建调拨单，明细包含：
- 商品A，批号P1，质量等级1
- 商品A，批号P2，质量等级1
- 商品B，批号P1，质量等级2
2. 审核入库		 系统按【商品编码、货主、批号、批次、质量等级】维度，生成3张独立的入库检验单 P0
TC-IN-003 检验单字段 验证检验单中从调拨单映射的字段 1. 生成检验单后，打开任意一张检验单详情
2. 核对以下字段：
- 验布来源 = “调拨入库”
- 入库单号 = 调拨入库单号
- 来源单号 = 原调拨单号
- 供应商 = 货主名称
- 仓库 = 调入仓
- 入库日期 = 调拨入库单的入库日期
- 到货质量等级 = 调拨明细中的质量等级
- 主/副数量 = 按维度汇总的调拨数量		 所有映射字段值均与调拨入库单明细信息一致，准确无误 P0
TC-IN-004 状态联动 生成检验单后，更新入库单和库存状态 1. 审核调拨入库生成检验单后
2. 查询该调拨入库单的“质检状态”
3. 查询对应库存记录的“质检状态”		 1. 入库单的质检状态变为“检验中”
2. 相关库存记录的质检状态变为“检验中”		 P0

三、 检验单操作模块测试

用例ID 模块 测试场景 测试步骤 预期结果 优先级
TC-INSP-001 检验操作 完成来源为“调拨入库”的检验单 1. 进入入库检验单列表，筛选出来源为“调拨入库”的检验单
2. 打开一张待检验的检验单
3. 录入检验数据，点击“完成检验”或“确认”按钮		 1. 操作流程与采购入库、调货入库的检验操作完全一致
2. 无报错，页面可正常流转
3. 检验单状态变更为“已确认”		 P0
TC-INSP-002 验布机集成 支持使用验布机对调拨入库检验单进行验布 1. 连接验布机设备并确保系统正常
2. 打开一张来源为“调拨入库”的检验单
3. 启动验布机数据采集流程		 验布机可正常接入，数据可回写至检验单，操作方式与现有采购/调货入库验布一致 P1

四、 检验后处理与数据更新模块测试

用例ID 模块 测试场景 测试步骤 预期结果 优先级
TC-UPDATE-001 库存更新 检验合格，更新库存质量信息 1. 将一张来源为调拨入库的检验单结果设为“合格”并确认
2. 根据生成时的维度（商品编码、货主、批号、批次、质量等级）查询库存记录		 1. 对应库存的“质检状态”更新为“合格”
2. 库存的“质量等级”、“质量异常类型”、“销售建议”、“销售建议说明”等字段按检验结果更新为合格相关信息		 P0
TC-UPDATE-002 库存更新 检验不合格，更新库存质量信息 1. 将一张来源为调拨入库的检验单结果设为“不合格”并确认
2. 根据生成时的维度查询库存记录		 1. 对应库存的“质检状态”更新为“不合格”
2. 库存的质量相关字段更新为不合格记录及原因		 P0
TC-UPDATE-003 入库单状态联动 一个入库单对应多张检验单，部分完成时，入库单状态保持“检验中” 1. 一个调拨入库单因多维度拆分，生成了2张检验单（INP1, INP2）
2. 完成INP1（合格或不合格）
3. 检查该入库单的“质检状态”		 入库单的质检状态仍为“检验中”，不会因为一张完成而变为最终状态 P0
TC-UPDATE-004 入库单状态联动 一个入库单对应多张检验单，全部合格完成 1. 延续TC-UPDATE-003，此时入库单状态为“检验中”
2. 完成INP2（设为合格）
3. 检查入库单质检状态		 入库单质检状态更新为“检验合格” P0
TC-UPDATE-005 入库单状态联动 一个入库单对应多张检验单，存在不合格时 1. 一个入库单对应INP1（合格）、INP2（不合格）
2. 全部完成确认
3. 检查入库单质检状态		 入库单质检状态更新为“检验不合格” P0
TC-UPDATE-006 异常场景 同维度调拨分两次入库，检验单完成时库存冲突 1. 场景：同一维度（商品A，货主X，批号P，批次B，等级1）的货物，通过两次独立的调拨入库到**同一个库位**
2. 因此生成两张检验单（INP-A, INP-B）
3. 先完成INP-A（库存更新）
4. 再完成INP-B		 1. 系统不崩溃，无异常抛出
2. INP-B完成时，由于对应“检验中”状态的库存已被INP-A更新，应给出友好提示（如“无可更新库存”）并记录日志
3. 不影响其他数据和流程		 P1

五、 UI显示与飞书消息通知模块测试

用例ID 模块 测试场景 测试步骤 预期结果 优先级
TC-UI-001 物理库存查询 显示“库存质检状态”列 1. 使用辰景、新全景、盛景任一主体账号登录
2. 进入【物理库存查询】页面
3. 查看列表表头		 列表表头中“库存质检状态”字段可见（之前对某些主体隐藏的列已去掉隐藏） P1
TC-MSG-001 飞书消息 检验结果为“已确认”时，推送消息到指定群 1. 完成一张来源为“调拨入库”的检验单并确认
2. 查看预配置的“仓库+品控群”飞书群聊		 群内收到一条【入库检验结果同步】消息，格式符合要求，包含以下关键字段：
- 检验单号
- 检验来源：调拨入库
- 入库单号、来源单号
- 供应商名称、商品编码、商品名称
- 批号（为空时显示为空）
- 入库主/副数量
- 检验结果（合格/不合格/免检）
- 品控实际检验时间、品控人员
- 检验工单完成时间		 P0
TC-MSG-002 飞书消息 消息推送的接收群组正确 1. 完成检验单
2. 检查消息接收群组设置		 消息仅推送到配置的“仓库+品控群”，不推送到其他无关群组 P1
TC-MSG-003 飞书消息 无批号时，消息中“批号”字段为空 1. 创建一条调拨入库检验单，该商品无批号信息
2. 完成该检验单
3. 查看飞书消息内容		 消息中“批号”字段显示为空字符串或预设占位符（如“--”），消息整体格式不错乱，无报错 P2
